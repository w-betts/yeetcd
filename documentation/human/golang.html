Module

Description

Go SDK for executing continuous deployment pipelines. Provides the runtime engine that assembles pipelines from protobuf definitions, builds container images, and executes work items using Docker, Kubernetes, or mock execution engines.

Contracts

  • Source extraction produces valid buildable source directories
  • Image building produces runnable images containing all artifacts
  • Pipeline assembly deserializes protobuf into executable pipeline objects
  • Work execution respects dependencies and conditions
  • Work results are tracked and shared with dependent work items
  • Failed work causes dependent work to be skipped (unless condition specifies otherwise)

Invariants

  • Work is executed at most once per pipeline run (idempotent execution via WorkResultTracker)
  • Work dependencies form a directed acyclic graph
  • Container images are built with unique tags based on source SHA
  • All work results are captured including stdout, output files, and status
  • Pipeline execution is complete when all final work items complete

Responsibilities

  • Provide runtime pipeline execution engine
  • Build container images from source code
  • Assemble pipelines from protobuf definitions
  • Execute work items in containers with proper dependency handling
  • Track work results and handle output mounts
  • Support multiple execution engines (Docker, Kubernetes, Mock)
  • Provide CLI for local development and testing
  • Manage work context and environment variable passing
