What is yeetcd.controller.source?

The yeetcd.controller.source package handles the extraction and building of source code submitted to yeetcd. It extracts source code from zip archives, parses the yeetcd.yaml configuration file, and builds the source code using containerized build environments.

This package is the first step in the pipeline execution process. Before any work can be executed, the source code must be extracted, configured, and built into a runnable state. The build process runs in a container to ensure reproducibility and isolation.

Why Does This Package Exist?

Source code handling requires careful management:

  • Zip Extraction: Safely extracting uploaded source archives
  • Configuration Parsing: Reading and validating yeetcd.yaml files
  • Containerized Builds: Running builds in isolated environments
  • Build Caching: Avoiding redundant builds for unchanged source
  • Artifact Management: Capturing build outputs for execution

Core Classes

SourceExtractor

Extracts source code from zip archives and parses the yeetcd.yaml configuration file. Validates that the extracted source contains a valid configuration and calculates a SHA for the source content.

SourceBuilder

Builds source code using the configured build image and command. Uses the ExecutionEngine to run builds in containers. Implements build caching based on source SHA to avoid redundant builds.

Source

Represents extracted source code ready for building. Contains the source directory path, the parsed yeetcd.yaml configuration, and a SHA hash for caching.

YeetcdDefinition

Parsed configuration from yeetcd.yaml. Defines the project name, language, build image, build command, and artifact definitions that should be captured after the build.

ArtifactDefinition

Defines a build artifact to capture. Specifies the artifact name and the path within the build container where the artifact can be found.

Contracts

Source extraction produces valid directories with yeetcd.yaml files
Build caching uses source SHA as cache key
Build commands are executed in isolated containers
All artifacts defined in yeetcd.yaml are captured after build

Invariants
