yeetcd Documentation

Continuous Deployment Solution with Container-Based Pipeline Execution

What is yeetcd?

yeetcd is a continuous deployment solution that enables you to define and execute deployment pipelines using container-based execution. It provides a Java SDK for defining pipelines programmatically, a controller for orchestrating execution, and protocol buffer definitions for language-agnostic pipeline serialization.

The system is designed around the concept of work items - discrete units of execution that can depend on other work items, run in containers, and pass data between each other. Pipelines are constructed using a fluent builder API and executed using Docker or Kubernetes.

Why yeetcd?

Traditional CI/CD systems often require complex YAML configurations and provide limited flexibility for dynamic pipeline generation. yeetcd addresses these limitations by:

  • Allowing pipelines to be defined in code using a type-safe Java API
  • Supporting dynamic work generation at runtime based on context
  • Enabling complex dependency graphs between work items
  • Providing first-class support for containerized execution
  • Offering a testing framework for unit testing pipelines without containers

The architecture follows a clear separation of concerns:

  • Protocol: Defines the language-agnostic protobuf schema for pipeline serialization
  • Java SDK: Provides the API for defining pipelines and generates protobuf output
  • Controller: Orchestrates source extraction, building, and pipeline execution
  • Java Test: Provides testing utilities for unit testing pipeline logic
