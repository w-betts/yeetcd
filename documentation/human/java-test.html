Module

Description

The java-test module provides testing utilities for unit testing pipeline definitions without requiring container execution. It offers a fake pipeline runner that simulates work execution in-memory, enabling fast, deterministic tests that can run without Docker or Kubernetes infrastructure.

This module is essential for test-driven development of pipelines. Rather than waiting for slow container-based tests, developers can use FakePipelineRunner to verify pipeline logic, dependency resolution, and condition evaluation in milliseconds.

Why java-test?

Testing pipelines traditionally requires:

  • Running Docker or Kubernetes infrastructure
  • Building container images
  • Waiting for actual work execution
  • Dealing with network and resource constraints

The java-test module eliminates these requirements by:

  • In-Memory Execution: Simulates work execution without containers
  • Configurable Results: Specify exactly what results each work item should produce
  • Fast Feedback: Tests run in milliseconds instead of minutes
  • Deterministic Behavior: Same inputs always produce same outputs
  • Production Logic: Uses the same condition evaluation logic as the real controller

Contracts

Deterministic Execution: FakePipelineRunner executes work deterministically based on specified results, ensuring tests are reproducible.
Production Condition Logic: Work conditions are evaluated using the same logic as production, ensuring test validity.
Dependency Respect: All work dependencies are respected during simulated execution, maintaining execution order.
Environment Capture: Test execution captures environment variables and input files as they would be in production.

Invariants

Dependency Order: Work execution order respects the dependency graph, ensuring prerequisites complete before dependents.
Result Caching: Completed work results are cached and reused, ensuring idempotent execution within a test run.
Unique Work Keys: Work keys are unique based on work and merged context, enabling proper identification.

Usage Example

Here's how to use FakePipelineRunner to test a pipeline:

@Test
void testPipeline() {
    // Create the pipeline under test
    Pipeline pipeline = createMyPipeline();
    
    // Configure the fake runner
    FakePipelineRunner runner = FakePipelineRunner.builder()
        .defaultWorkResult(FakeWorkResult.builder()
            .status(FakeWorkStatus.SUCCESS)
            .stdOut("Build successful")
            .build())
        .build();
    
    // Run the pipeline
    FakePipelineRunResult result = runner.run(
        FakePipelineRun.builder(pipeline)
            .arguments(Map.of("VERSION", "1.0.0"))
            .build()
    );
    
    // Assert results
    assertEquals(FakePipelineStatus.SUCCESS, result.getStatus());
    assertTrue(result.getWorkExecutions().size() > 0);
}

Testing Philosophy

The java-test module enables a testing pyramid approach for pipelines:

  • Unit Tests: Test individual work definitions and conditions in isolation
  • Integration Tests: Test complete pipelines with FakePipelineRunner
  • End-to-End Tests: Run actual pipelines with the controller (slower but necessary)

By using FakePipelineRunner for the majority of tests, you get fast feedback during development while still having confidence that your pipeline logic is correct.
