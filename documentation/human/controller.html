Module

Description

The controller module is the runtime engine that builds and executes pipelines. It orchestrates the complete pipeline lifecycle from source extraction through image building, pipeline assembly from protobuf definitions, and work execution using Docker or Kubernetes execution engines.

This module bridges the gap between the declarative pipeline definitions created with the Java SDK and the actual execution environment. It handles all the complexity of container management, dependency resolution, and result tracking.

Why Controller?

The controller provides a complete runtime environment for pipeline execution:

  • Source Management: Extracts source code from zip archives and manages build environments
  • Image Building: Creates container images containing pipeline definitions and artifacts
  • Dependency Resolution: Ensures work items execute in the correct order based on dependencies
  • Condition Evaluation: Determines whether work should execute based on context and previous results
  • Result Tracking: Captures work outputs and makes them available to dependent work items
  • Multi-Engine Support: Supports both Docker and Kubernetes execution environments

Responsibilities

  • Extract and build source code from zip archives
  • Build container images containing pipeline definitions
  • Assemble executable pipelines from protobuf output
  • Execute pipeline work items in containers
  • Manage work dependencies and execution order
  • Track work results and handle dynamic work generation
  • Support multiple execution engines (Docker, Kubernetes)

Dependencies

  • protocol
  • java-sdk (for protobuf deserialization)

Packages

yeetcd.controller.source

Source code extraction and building. Handles extracting source code from zip archives, parsing yeetcd.yaml configuration files, and building source code using configured build images and commands.

yeetcd.controller.pipeline

Core pipeline execution and orchestration classes. Manages the runtime representation of pipelines, work execution, result tracking, and condition evaluation. Bridges the gap between deserialized protobuf definitions and actual container execution.

yeetcd.controller.execution

Execution engine abstraction for running containerized jobs. Provides interfaces and implementations for building container images and executing jobs using Docker or Kubernetes. Handles image building, job execution, log streaming, and output extraction.

Contracts

Valid Source Extraction: Source extraction produces valid buildable source directories with all necessary files.
Runnable Images: Image building produces runnable images containing all artifacts required for execution.
Correct Deserialization: Pipeline assembly deserializes protobuf into executable pipeline objects with all work definitions properly mapped.
Dependency Respect: Work execution respects dependencies and conditions, ensuring work items run in the correct order.
Result Sharing: Work results are tracked and shared with dependent work items, including stdout and output files.
Failure Propagation: Failed work causes dependent work to be skipped unless the condition specifies otherwise.

Invariants

Idempotent Execution: Work is executed at most once per pipeline run via WorkResultTracker.
Acyclic Dependencies: Work dependencies form a directed acyclic graph, preventing circular dependencies.
Unique Image Tags: Container images are built with unique tags based on source SHA for reproducibility.
Complete Result Capture: All work results are captured including stdout, output files, and status.
Execution Completion: Pipeline execution is complete when all final work items complete, regardless of success or failure.

Execution Flow

The controller follows a clear execution flow:

  1. Source Extraction: Extract source code from the provided zip archive to a temporary directory
  2. Configuration Parsing: Parse all yeetcd.yaml files to understand project structure and build requirements
  3. Source Building: Execute build commands in containerized environments to produce artifacts
  4. Image Building: Create container images containing the built artifacts and pipeline definitions
  5. Pipeline Assembly: Run the generated pipeline definitions class to output protobuf, then deserialize it
  6. Work Execution: Execute work items in dependency order, respecting conditions and tracking results
  7. Result Aggregation: Collect all work results and produce the final pipeline result
