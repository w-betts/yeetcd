What is yeetcd.sdk.condition?

The yeetcd.sdk.condition package provides a flexible condition system for controlling when work items should execute within a pipeline. Conditions enable dynamic, context-aware pipeline execution where work items can be skipped based on runtime conditions, previous work results, or work context values.

This package implements the Specification Pattern combined with Composite Pattern to allow complex logical combinations of conditions. Developers can create simple conditions and combine them using AND, OR, and NOT operators to express sophisticated execution rules.

Why Does This Package Exist?

Real-world deployment pipelines often need conditional execution logic:

  • Environment-Specific Work: Only run certain steps in production vs. staging
  • Failure Handling: Run cleanup or rollback steps only when previous work fails
  • Parameter-Driven Flow: Execute different branches based on pipeline parameters
  • Dynamic Pipelines: Skip expensive operations when not needed

Without a condition system, developers would need to embed this logic within the work itself, making pipelines harder to understand and test. The condition package separates execution control from execution logic, improving clarity and maintainability.

Core Classes

Condition (Abstract Base)

The foundation of the condition system. All conditions extend this class and implement the evaluate() method which receives the current work context and previous work results, returning a boolean indicating whether the condition is satisfied.

Logical Composition Conditions

AndCondition: All contained conditions must evaluate to true.
OrCondition: At least one contained condition must evaluate to true.
NotCondition: Negates the result of a contained condition.

WorkContextCondition

Evaluates work context key-value pairs. Used to check if specific context values match expected values, enabling parameter-driven pipeline flows.

PreviousWorkStatusCondition

Checks the execution status of previous work items (SUCCESS, FAILURE, SKIPPED). Essential for implementing failure handling and conditional rollback logic.

Conditions (Factory)

Static factory class providing convenient methods for creating all condition types. Offers a fluent API for building complex condition hierarchies.

Usage Example

import static yeetcd.sdk.condition.Conditions.*;

// Only run in production environment
Work prodWork = Work.builder("deploy", deployDef)
    .condition(workContextCondition("ENV", EQUALS, "production"))
    .build();

// Run cleanup only if previous work failed
Work cleanup = Work.builder("cleanup", cleanupDef)
    .condition(previousWorkStatusCondition(FAILURE))
    .build();

// Complex condition: Run if (A AND B) OR (C AND NOT D)
Condition complex = orCondition(
    andCondition(condA, condB),
    andCondition(condC, notCondition(condD))
);

Contracts

Conditions are immutable after construction
Logical conditions short-circuit evaluation where possible
All conditions must be serializable to protobuf
Work context conditions support equality matching only

Invariants

Condition evaluation is deterministic for same inputs
Logical conditions maintain left/right associativity
No circular references in composed conditions

Responsibilities

  • Define Condition base class for all conditions
  • Implement logical composition conditions (And, Or, Not)
  • Implement work context conditions for key-value matching
  • Implement previous work status conditions
  • Serialize conditions to protobuf format
